Question 3 communication, comparing encoded data

From our discussions, we listed several ways we can compare visually-encoded data, from more effective to less effective.

From the Centers Activity and Balance graphic, what type(s) of visual comparisons do the encodings enable? Explain to a mixed audience of Citi Bike analytics executives.

The Centers graphic uses heatmap data visualization, graphical representations of data that utilize color-coded systems for comparison. The more bikes arrive at the destination, the higher the color chroma or intensity of orange color. In contrast, the fewer bikes arrive at the destination, the low chroma or intensity of oranges color and eventually, the orange color fades and become dark color. Moreover, the more bikes leave at origin, the high chroma or intensity of blue color. On the other hand, the fewer bikes leave at the origin, the low chroma or intensity of oranges color and eventually the blue color fade and become dark color.The activity gradient shows the number of bikes movement of arrival and departure. More traffic at the station, the lighter color area. The less traffic at the station, the darker the color area. For example, if we look at the first address on the y-axis of 1 Ave& E 15 St, and 8 AM on X-axis, the color is bright orange. The color suggests that many bikes arrive at this station, and at 8 AM is a busy time as people travel to work.

To decide on the color, we calculate the difference between the number of bikes leave and the number of bikes that arrive. Moreover, the absolute value of the difference can indicate the degree of chroma and luminance. The lighter the color, the higher the value; the darker the color, the lower its value. White is the color with the highest value; black is the color with the lowest value. If the difference is positive, it suggests the origin station and shows as blue color. If the difference is negative, it indicates the destination station with orange color. If the difference is zero, no bike arrives or leaves and shows the darker color. Finally, if the difference is highly positive (only bike arrival), this indicates a lighter area.

We need to change the structure of our data so that we can map data values onto the visual encodings used in the Centers graphic.

More specifically, we need to know the number of rides both starting and ending at each station name at each hour of the day, averaged over the number of days in our data set. Well need to create new variables and pivot some of the data. Specifically, we will create a variable for day of month (day) and hour of day (hour) from the existing variable starttime. Then, we will pivot two variables start_station_name and end_station_name into long format, like so:

df <- 
  df %>%
  mutate(
    day = format(starttime, "%d"),
    hour = format(starttime, "%H")
  ) %>%
  pivot_longer(
    cols = c(start_station_name, end_station_name), 
    names_to = "start_end",
    values_to = "station_name"
  ) %>%
  mutate(
    station_name = fct_reorder(station_name, desc(station_name))
  )

The pivot results in creating separate observations, from the perspective of a docking station (instead of the perspective of a ride), for both types of events: a bike parking and a bike leaving.

Question 5 workflow, transforming data

With the pivoted data frame, we can now group our data by station name and hour, and calculate the averages well need to map onto visual variables.

Create new variables activity and balance, where activity holds the average number of rides or observations at each station name each hour and where balance hold the average difference between rides beginning at the station and rides ending at the station.

df <- 
  df %>%
  group_by(station_name, hour, .drop = FALSE) %>%
  
  summarise (
    activity = n(),
    balance  = sum(if_else(start_end == 'start_station_name', -1, +1) ) 
  ) %>%
  
  ungroup()

Question 7 workflow, mapping data to visual channels

Finally, we are ready to map our data onto the visual variables. The Centers Activity and Balance graphic resembles a so-called heatmap.

Use the grammar of graphics to create tiles of information, using the function geom_tile. To do that, first review the help file for that function, paying particular attention to the aesthetics youll need to specify.

Further, to map the individual channels of color, you can use the function hcl thats already loaded from tidyverse, which works very similarly to (a bit less optimal than) the example I showed you from my R package, hsluv_hex. You may also use mine, but that will require you to install it.

Ive started the code for you below. Add code where prompted.

p <- 
  df %>%
  ggplot() +
  scale_fill_identity() +
  geom_tile(
    mapping = aes(
      x = hour,
      y = station_name,
      fill = hcl(hue, saturation, luminance)
    ),
    width = 0.95,
    height = 0.95
  ) +
  theme_dark() +
  theme(
    panel.background = element_blank(),
    panel.grid = element_blank(),
    plot.background = element_rect(fill = "#333333"),
    axis.text.x = element_text(color = "#888888", size = 16 / .pt),
    axis.text.y = element_text(color = "#888888", size =  7 / .pt)
  ) +
  labs(x = "", y = "")

How might 1) empty and full docking stations and 2) CitiBike rebalancing bikes affect the visual patterns in our graphic? Explain to a Citi Bike analytics executive.

From the above argument, we define activity as the average number of rides at each station name each hour and where balance is the average difference between rides beginning at the station and rides ending at the station. An empty station means no bikes enter and park at the station, and only the bikes leave at the station. As a result, the station has no bikes left and becomes empty. As a result, the average number of rides or activities at the station is high. However, the average difference between rides beginning at the station and rides ending at the station or balance is negative. The virtual pattern will show the black/dark color = HCL(50, 100, 0). In contrast, a full station means there are full of bikes as no bikes leave the station. As a result, the average number of rides or activities at the station is zero. However, the average difference between rides beginning at the station and rides ending at the station or balance is highly positive. The virtual pattern will show the white/bright color = HCL(200, 0, 100). Rebalancing means restoring the bikes from an empty station to the full station and optimizing availability and drop-off. As a result, the activities at the station are positive. However, the balance is zero because when the bikes leave the station, new bikes enter the station to rebalance. The virtual pattern will show the black/dark color = HCL(0, 100, 0).
